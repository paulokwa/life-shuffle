## Current Onboarding Finding

Firestore currently has `displayNameConfirmed: true`, a display name present, `calendarNameConfirmed: true`, `introOnboardingCompleted: true`, and title `Kwame and Laura` for the inspected calendars. Clearing browser/client data does not reset those remote onboarding fields.

The original audit found a mismatch: mini onboarding completion was widget-local only, so reloads showed it again. That is now fixed by persisting `introOnboardingCompleted` locally and in the flat Firestore `SavedState` map. The inspected remote calendars now report `introOnboardingCompleted: true`, so returning signed-in users should skip the intro even after browser/client data is cleared.

Expected behavior after these fixes: when signed in and Firestore data exists, the app shows a small loading state while initial remote sync completes, then skips display-name, calendar-name, and mini intro onboarding if Firestore already confirms them. Settings > Privacy/help includes `Replay intro` for manual replay without resetting the completed state.
